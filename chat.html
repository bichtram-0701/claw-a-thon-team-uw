Product pitch

Funnel Agent turns funnel drift into owned recovery work.

Business teams already have dashboards, Jira, Confluence, and Teams. The gap is not visibility. The gap is converting metric movement into ranked business risk, ownership, action, meeting memory, and follow-up.

What is Funnel Agent?

Execution agentIt connects funnel metrics to Jira work, Confluence decisions, and Teams notifications.
Business funnel layerIt understands Traffic -> Submission -> Approval -> Disbursement, targets, owners, drop reasons, and value at risk.
Workflow systemIt follows one loop: Detect -> Diagnose -> Assign -> Summarize -> Notify.

What problem does Funnel Agent solve?

Problem
Dashboards show that a metric moved, but managers still have to decide which movement matters, who owns it, whether recovery work exists, and what should happen next.
Bottleneck
The real bottleneck is metric-to-action translation: turning drift into accountable recovery work fast enough to recover the target.
Solution
Funnel Agent queries the funnel data, computes metrics, ranks the risk, explains the evidence, defaults the owner from the Jira Epic -> stage owner structure, updates the recovery task, prepares the Confluence weekly readout, and posts blockers to Teams.

Why agent, not normal ChatGPT?

Normal chatAnswers a question if the user brings the context.
Funnel AgentRuns a governed workflow across metrics, Jira, Confluence, and Teams.
Safety principleThe LLM explains. Deterministic tools compute metrics, validate SQL templates, deduplicate Jira writes, and publish notifications.

How the workflow systems fit together

JiraExecution system of record. Each stage has an Epic, an operational stage owner, and tasks. If no assignee is named, Funnel Agent defaults to the stage owner.
ConfluenceMeeting memory. Weekly pages capture metric signals, blocker context, decisions, owners, and next recovery actions.
TeamsNotification layer. It can post off-track blockers, Jira task updates, due-soon reminders, and stale-task digests.

FAQ

Is this just a dashboard?
No. A dashboard stops at reporting. Funnel Agent connects the metric to value at risk, owner, blocker context, Jira action, Confluence readout, and Teams follow-up.
Is this just Jira AI?
No. Jira knows work items; it does not know the funnel target gap, value-at-risk formula, drop-reason reconciliation, or weekly metric recovery loop.
Does the LLM decide the numbers?
No. Query-backed Python/SQL computes metrics, MoM changes, value at risk, drop-reason breakdowns, and write guards. The model routes, extracts bounded fields, and explains verified facts.
Does it prove causality?
No. The diagnostic layer shows contribution analysis and reconciliation. It says where the drop is concentrated, not that a reason caused the decline.
Why no slash command for funnel questions?
Funnel questions are read-only intelligence, so users can ask naturally. Slash commands are reserved for external tool boundaries: /jira, /confluence, and /teams. Advanced /query is still available if someone explicitly wants raw/audit SQL.
What is defensible?
The Funnel Execution Graph: metric -> target -> stage -> owner -> initiative -> blocker -> decision -> outcome. Over time, this becomes team-specific operating memory.
What is still future work?
Closed-loop outcome learning: when a Jira recovery task is completed, compare before/after metrics and classify whether the action worked, partially worked, or was inconclusive.
