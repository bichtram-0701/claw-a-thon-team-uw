# Conversion Funnel Analyst 🤖

**GreenNode Claw-a-thon 2026 — Track: Data Analysis**

> 🤖 This is an AI agent. Every report it produces is clearly marked as
> AI-generated to end users (per Rulebook §11.1).

An AI agent that **automatically retrieves funnel/event data, analyzes the
conversion path stage-by-stage, and synthesizes a decision-ready report** with
prioritised recommendations — turning a raw CSV into an executive brief without
anyone running the analysis by hand.

---

## Problem · User · Solution

- **Problem.** Conversion-funnel analysis is repetitive and manual: pull the
  data, compute conversion at each step, find the biggest drop-off, slice by
  segment, then write it up. It eats analyst hours and is done inconsistently.
- **User.** Growth, product, and ops analysts who own a conversion funnel —
  signup, checkout, onboarding, loan application, and similar.
- **Solution.** One call → the agent computes the funnel deterministically
  (counts, step conversion, drop-off, velocity, segment breakdowns) and an LLM
  (Gemma/Qwen via GreenNode MaaS) writes the executive summary and ranked,
  numbers-backed recommendations.
- **Value.** Minutes instead of hours, consistent reporting, and the
  highest-impact leak surfaced automatically.

## How it works — retrieve → analyze → synthesize → report

```
CSV (synthetic) ─▶ RETRIEVE ─▶ ANALYZE (deterministic) ─▶ SYNTHESIZE (LLM) ─▶ report.md
```

The math is deterministic and auditable; the LLM only writes up the **verified**
numbers, so the report never contains hallucinated figures. The agent
self-verifies its metrics (counts monotonic, conversion+drop-off = 100%, segment
totals reconcile) before writing.

## Quick start

```bash
# 1. (optional) generate the synthetic demo dataset — already bundled
python scripts/generate_sample_data.py

# 2. run on the bundled synthetic data (works with NO API key — uses fallback)
python run.py --out report.md

# 3. full AI narrative: set MaaS credentials, then run
cp .env.example .env        # fill in LLM_API_KEY (and base URL / model)
export $(grep -v '^#' .env | xargs)
python run.py --data data/sample_funnel.csv --out report.md
```

No third-party packages are required — the agent uses the Python standard
library (`urllib`, `csv`, `json`) and an OpenAI-compatible call to MaaS.

## Model (rulebook-compliant)

Uses the contest-provided self-hosted MaaS models **`gemma-4-31b-it`** or
**`Qwen-3-27B`** through GreenNode's OpenAI-compatible API — free under the POC
wallet. All LLM settings are environment variables (`LLM_BASE_URL`,
`LLM_API_KEY`, `LLM_MODEL`), so the same code targets the public MaaS endpoint or
your self-hosted OpenClaw instance URL from the welcome email. No external paid
model is used.

## Data compliance (important)

Per the Claw-a-thon Tool Access Guideline, this project uses **100% synthetic
data only** — no real customer data, no PII, no confidential/restricted internal
data. The bundled `data/sample_funnel.csv` is generated by a seeded script with
fake user IDs. If you point the agent at your own file, it must likewise be
synthetic, public, or anonymized. See `COMPLIANCE.md`.

## Use your own funnel

Edit `config.json` to declare your stages (each with a timestamp column) and
segment dimensions, then run with `--config`. The engine is stage-agnostic — it
works for any ordered funnel where each stage has a timestamp column.
