# Compliance notes

This project is built to satisfy the GreenNode Claw-a-thon 2026 *Official
Rulebook (v1.2)* and the *Support & Tool Access Guideline*.

## Data (hard rule)

> The product may use **public, synthetic, or anonymized** data only. It must
> **NOT** use real customer data, personally identifiable information (PII), or
> confidential / restricted internal data — *even after tool access is granted.*

- All bundled data (`data/sample_funnel.csv`) is **synthetic**, generated by a
  seeded script (`scripts/generate_sample_data.py`) with fabricated IDs.
- The agent reads only the file it is pointed at; the README and CLI help both
  state that any user-supplied file must also be synthetic/public/anonymized.
- The core demo needs **no internal tool access**, so no IT Helpdesk request is
  required. If a later version connects to Office 365 (Teams/Outlook/SharePoint),
  the request goes to **helpdesk@vng.com.vn** (8h first-response SLA), scoped to
  **personal** resources only — see `EMAIL_TEMPLATE_office365.md`.

## AI disclosure (Rulebook §11.1)

Every report begins with a fixed AI-disclosure banner (see
`src/prompts.py → AI_DISCLOSURE`). The agent never impersonates a human.

## Model declaration

Uses GreenNode MaaS self-hosted models (`gemma-4-31b-it` / `Qwen-3-27B`) via the
OpenAI-compatible API. No external paid model is used. If you switch to an
external model (OpenAI/Anthropic), you must declare it here and bear the cost
(Rulebook FAQ).
